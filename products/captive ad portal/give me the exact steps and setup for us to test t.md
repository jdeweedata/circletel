# give me the exact steps and setup for us to test this configuration in a lab environment before we deploy it to site

Here is a complete, step-by-step lab guide based on the exact Powerlynx + MikroTik VLAN documentation and configuration patterns confirmed from the search results.

## Lab Equipment You Need

| Device | Purpose | Minimum Spec |
| :-- | :-- | :-- |
| MikroTik hAP ac² or hAP ax² | Main lab router (matches site hardware) | RouterOS 7.x |
| Laptop/PC (x2) | One simulating nurse, one simulating patient | Any |
| Smartphone | Test patient captive portal UX | Any |
| Internet uplink | Simulate ECHO BNG (your home/office router) | Any broadband |
| Powerlynx account | Free 21-day trial | cloud.powerlynx.app |
| WinBox | MikroTik GUI management | Download from mikrotik.com |

> You do **not** need Interstellio in the lab — you'll simulate the nurse VLAN with a simple WPA2 password since Interstellio is only authenticating real staff credentials. The VLAN isolation and bandwidth separation is what you're testing.

***

## Phase 1 — MikroTik Base Setup

### Step 1.1 — Factory Reset \& Update

Connect to MikroTik via WinBox (MAC address login). Run:

```
/system reset-configuration no-defaults=yes skip-backup=yes
```

After reboot, update RouterOS:

```
/system package update download
/system reboot
```


### Step 1.2 — Create the Bridge with VLAN Filtering

```
/interface bridge
add name=bridge-main vlan-filtering=yes
```


### Step 1.3 — Add Physical Ports to Bridge

```
/interface bridge port
add bridge=bridge-main interface=ether2 pvid=1
add bridge=bridge-main interface=ether3 pvid=1
add bridge=bridge-main interface=ether4 pvid=1
```


### Step 1.4 — Create the Two VLAN Interfaces

```
/interface vlan
add interface=bridge-main name=vlan10-nurse vlan-id=10
add interface=bridge-main name=vlan20-patient vlan-id=20
```


### Step 1.5 — Register VLANs on the Bridge

```
/interface bridge vlan
add bridge=bridge-main tagged=bridge-main vlan-ids=10
add bridge=bridge-main tagged=bridge-main vlan-ids=20
```


### Step 1.6 — Create Two Wireless SSIDs

On RouterOS 7 with built-in Wi-Fi:

```
/interface wifiwave2
set [ find default-name=wifi1 ] configuration.ssid="CircleTel-Clinic-Staff" \
  configuration.mode=ap security.authentication-types=wpa2-psk \
  security.passphrase="StaffSecurePass" disabled=no

add master-interface=wifi1 name=wifi-patient \
  configuration.ssid="CircleTel-Clinic-Free" \
  configuration.mode=ap security.authentication-types="" disabled=no
```


### Step 1.7 — Assign SSIDs to VLANs

```
/interface bridge port
add bridge=bridge-main interface=wifi1 pvid=10
add bridge=bridge-main interface=wifi-patient pvid=20
```


***

## Phase 2 — IP Addressing \& DHCP

### Step 2.1 — Assign IPs to VLAN Interfaces

```
/ip address
add address=192.168.10.1/24 interface=vlan10-nurse
add address=192.168.20.1/24 interface=vlan20-patient
```


### Step 2.2 — Create DHCP Pools

```
/ip pool
add name=pool-nurse ranges=192.168.10.10-192.168.10.254
add name=pool-patient ranges=192.168.20.10-192.168.20.254
```


### Step 2.3 — Create DHCP Servers

```
/ip dhcp-server
add address-pool=pool-nurse interface=vlan10-nurse name=dhcp-nurse
add address-pool=pool-patient interface=vlan20-patient name=dhcp-patient

/ip dhcp-server network
add address=192.168.10.0/24 gateway=192.168.10.1 dns-server=8.8.8.8
add address=192.168.20.0/24 gateway=192.168.20.1 dns-server=8.8.8.8
```


### Step 2.4 — WAN / Simulated ECHO BNG

Plug `ether1` into your lab internet (home router). In a real deployment this is your ECHO PPPoE — in the lab, DHCP is fine:

```
/ip dhcp-client
add interface=ether1 disabled=no
```


***

## Phase 3 — Firewall \& NAT

### Step 3.1 — NAT Masquerade (Both VLANs Get Internet)

```
/ip firewall nat
add chain=srcnat out-interface=ether1 action=masquerade
```


### Step 3.2 — Inter-VLAN Isolation (Nurses Can't See Patients)

```
/ip firewall filter
add chain=forward in-interface=vlan20-patient out-interface=vlan10-nurse \
  action=drop comment="Block patients from nurse VLAN"
add chain=forward in-interface=vlan10-nurse out-interface=vlan20-patient \
  action=drop comment="Block nurses from patient VLAN"
```


***

## Phase 4 — Powerlynx Captive Portal (Patient VLAN Only)

### Step 4.1 — Register on Powerlynx

Go to `cloud.powerlynx.app` → Create account → Start free trial → **Add New Hotspot** → Select **MikroTik Advanced Setup**.

Note down:

- Your **WireGuard public key** (generated by Powerlynx)
- Your **NAS IP** (assign as `192.168.20.1`)
- Your **RADIUS secret** (you set this)


### Step 4.2 — Configure WireGuard Tunnel on MikroTik

```
/interface wireguard
add name=wg-powerlynx listen-port=13231 private-key="<generated-key>"

/interface wireguard peers
add interface=wg-powerlynx \
  public-key="<powerlynx-public-key>" \
  endpoint-address="<powerlynx-server-ip>" \
  endpoint-port=13231 \
  allowed-address=172.16.0.0/30 \
  persistent-keepalive=25s

/ip address
add address=172.16.0.2/30 interface=wg-powerlynx
```

Paste your MikroTik's WireGuard **public key** into the Powerlynx dashboard [^1].

### Step 4.3 — Add Powerlynx as RADIUS Client

```
/radius
add address=172.16.0.1 secret="<your-radius-secret>" \
  service=hotspot src-address=192.168.20.1 timeout=3000ms
```

The `src-address` must match the **NAS IP** you entered in Powerlynx [^2].

### Step 4.4 — Run Hotspot Setup on Patient VLAN Only

In WinBox go to **IP → Hotspot → Hotspot Setup** and when prompted for interface, select **`vlan20-patient`** only — never the bridge or nurse VLAN [^3].

Follow the wizard:

- Local address: `192.168.20.1`
- Address pool: use `pool-patient`
- DNS: `8.8.8.8`
- SMTP: skip
- DNS name: `clinic-wifi.local` (for portal redirect)


### Step 4.5 — Enable RADIUS on the Hotspot Server

```
/ip hotspot
set [find name=hotspot1] use-radius=yes
```


### Step 4.6 — Walled Garden (Allow Splash Page Pre-Auth)

```
/ip hotspot walled-garden
add dst-host="*.powerlynx.app"
add dst-host="cloud.powerlynx.app"
add dst-host="*.googleapis.com"
add dst-host="*.gstatic.com"
```

This allows unauthenticated patients to reach the splash page before they've accepted T\&Cs [^4].

## Phase 5 — Bandwidth Splitting (Simulate the 50Mbps)

### Step 5.1 — Queue Tree to Split the Link

```
/queue tree
add name=WAN-Total parent=ether1 max-limit=50M

add name=Nurse-Queue parent=WAN-Total \
  packet-mark=nurse-traffic max-limit=20M priority=1

add name=Patient-Queue parent=WAN-Total \
  packet-mark=patient-traffic max-limit=30M priority=8
```


### Step 5.2 — Mangle Rules to Mark Traffic

```
/ip firewall mangle
add chain=forward in-interface=vlan10-nurse \
  action=mark-packet new-packet-mark=nurse-traffic passthrough=no

add chain=forward in-interface=vlan20-patient \
  action=mark-packet new-packet-mark=patient-traffic passthrough=no
```


### Step 5.3 — Per-Patient Speed Cap (Powerlynx Side)

In the Powerlynx dashboard under your hotspot → **Plans** → create a free plan:

- Download: `5 Mbps`
- Upload: `2 Mbps`
- Session duration: `60 minutes`
- Daily data cap: `500 MB` (optional)

This is enforced via RADIUS attributes returned to MikroTik per session [^5].

***

## Phase 6 — Lab Testing Checklist

**Test 1 — VLAN Isolation**

- [ ] Nurse laptop (VLAN 10) gets `192.168.10.x` IP ✓
- [ ] Patient device (VLAN 20) gets `192.168.20.x` IP ✓
- [ ] Nurse cannot ping `192.168.20.x` range ✓
- [ ] Patient cannot ping `192.168.10.x` range ✓

**Test 2 — Captive Portal Flow**

- [ ] Connect phone to `CircleTel-Clinic-Free` SSID ✓
- [ ] Browser auto-redirects to Powerlynx splash page ✓
- [ ] Accept T\&Cs → internet access granted ✓
- [ ] Session timer counts down in Powerlynx dashboard ✓

**Test 3 — Bandwidth**

- [ ] Run speedtest on nurse VLAN — confirm ~20Mbps ✓
- [ ] Run simultaneous speedtest on patient VLAN — confirm capped at 5Mbps per device ✓
- [ ] Nurse speed unaffected when patients are active ✓

**Test 4 — RADIUS Logging**

- [ ] In Powerlynx dashboard → Logs → confirm patient MAC, session start/stop, data used ✓
- [ ] Disconnect a patient device → session terminates in dashboard ✓

**Test 5 — Ad/Splash Page**

- [ ] Upload clinic branding to Powerlynx splash page ✓
- [ ] Test on Android and iPhone — both redirect correctly ✓
- [ ] HTTPS sites show portal (requires DNS-based redirect, verify walled garden) ✓

Once all 5 test phases pass cleanly in the lab, the exact same RouterOS commands export directly to the site MikroTik — the only changes for production are the WAN interface (`ether1` becomes your Tarana/ECHO PPPoE interface) and the RADIUS secret matching Interstellio for the nurse VLAN.
