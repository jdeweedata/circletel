Order Flow — Step-by-Step Friction Audit

6 screens a customer touches, with friction points called out

1
Coverage Check /order/coverage
Address autocomplete + Residential / Business toggle → leads to /order/packages/[leadId]
✓ Clean
↓
2
Package Selection /order/packages/[leadId]
Grid of package cards with speed/price/features. Fibre & Wireless tabs.
⚠️ Mock Data
Friction: Packages are hardcoded (HomeFibre Basic R599→R399). Not fetched from the coverage/package API by leadId. Customer sees fake promo pricing.
↓
3
Account Creation /order/account
Name, email, password, phone + Google OAuth + terms checkbox
🔴 Flow Broken
Critical: On signup, user is redirected to /order/verify-otp for phone OTP — breaking them out of the order wizard. After OTP, unclear if they return to the order or land on dashboard. Customer likely abandons here.
↓ (OTP detour → /order/verify-otp → ???)
4
Service Address /order/service-address
Confirm installation address (read-only, from step 1) + select property type
⚠️ Redundant Screen
Friction: Address is read-only (already collected in step 1). This screen exists only to select property type (apartment/house/etc). Could be merged into step 1 or step 3 to remove a whole page.
"Create Order" button is here — not on the payment page. Order record is created before payment. This is confusing: customer expects "pay" = order created, but order is already in DB at this point.
↓
5
Payment /order/payment → NetCash
R1.00 validation charge → redirect to NetCash → return to confirmation
⚠️ UX Confusion
Friction: Page shows duplicate order creation logic (also calls /api/orders/create). Order may already exist from step 4, but payment page tries to create it again — duplicate order risk.
↓
✓
Confirmation /order/confirmation/[orderId]
Order summary + KYC status + "What happens next" steps
🔴 Shows Stale Data
Critical: Page reads from OrderContext (in-memory/localStorage), not from the database. If user refreshes or returns later, data is stale or gone. No live API fetch for the confirmed order.

Critical
OTP detour breaks order wizard mid-flow → likely high abandonment
