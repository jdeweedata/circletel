# Code Review Graph

Token-efficient knowledge graph for AI-assisted code reviews and blast-radius analysis.

**Repository**: [tirth8205/code-review-graph](https://github.com/tirth8205/code-review-graph)
**Version**: 2.2.1
**License**: MIT

---

## Why

CircleTel is a 254-page Next.js app with 2,557+ source files. When Claude Code reviews a PR or investigates an issue, it needs to read relevant files. Without a knowledge graph, it reads entire files or guesses which files matter.

Code-review-graph solves this by:
- Building a dependency graph of all symbols (functions, classes, imports)
- Exposing 22 MCP tools that Claude Code queries automatically
- Reducing token consumption by ~6.8x on code reviews
- Providing automated blast-radius analysis (aligns with our "blast radius minimization" principle)

---

## Installation

### Prerequisites

- Python 3.10+ (system has Python 3.11)

### Install the Package

```bash
pip3 install code-review-graph
```

### MCP Configuration

The `.mcp.json` file in the project root configures Claude Code to use the graph:

```json
{
  "mcpServers": {
    "code-review-graph": {
      "command": "code-review-graph",
      "args": ["serve"],
      "type": "stdio"
    }
  }
}
```

This was auto-generated by `code-review-graph install --platform claude-code`.

## Usage

### Build the Graph (First Time / After Major Changes)

```bash
code-review-graph build
```

Parses all files using Tree-sitter AST parsing. Takes ~30 seconds on CircleTel's codebase.

### Incremental Update (Daily Use)

```bash
code-review-graph update
```

Only re-parses changed files since last build. Completes in <2 seconds.

### Check Status

```bash
code-review-graph status
```

Shows node/edge counts, languages detected, last update time.

### Watch Mode (Auto-Update on File Changes)

```bash
code-review-graph watch
```

Monitors filesystem and updates graph automatically.

## How It Works with Claude Code

Once the MCP server is configured, Claude Code automatically discovers and uses the graph tools during sessions. No manual invocation needed.

### Key MCP Tools Available

| Tool | Purpose |
|------|---------|
| `get_symbols` | List functions, classes, imports in a file |
| `get_blast_radius` | Show which files/symbols are impacted by changes |
| `search_symbols` | Semantic search across the codebase |
| `get_dependencies` | Trace upstream/downstream dependencies |
| `get_file_summary` | Quick overview of a file's structure |
| `get_community` | Show related files grouped by functionality |

### Token Efficiency

Instead of reading entire files, Claude Code can:
1. Query the graph for relevant symbols
2. Read only the specific functions/classes needed
3. Understand impact of changes without reading the entire codebase

## Integration with CircleTel Workflow

### Blast Radius Analysis

Aligns with CircleTel's "blast radius minimization" principle from `CLAUDE.md`:

```bash
# Before making changes, check impact
code-review-graph detect-changes
```

### PR Reviews

The Claude Code Action (`.github/workflows/claude-code-review.yml`) benefits automatically - when reviewing PRs, Claude Code queries the graph to understand change impact.

### Session Startup

Consider running `code-review-graph update` as part of session startup to ensure the graph is current.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `command not found` | Run `pip3 install code-review-graph` |
| Stale graph | Run `code-review-graph update` or `code-review-graph build` |
| MCP not connecting | Verify `.mcp.json` exists in project root, restart Claude Code |
| Missing language support | Tree-sitter supports 19 languages; TypeScript/TSX/JS/Python are all covered |

## Maintenance

- **Rebuild** after major refactors or branch switches: `code-review-graph build`
- **Update** daily or before code reviews: `code-review-graph update`
- **No upgrades needed** for the graph database - migrations run automatically
- **Package updates**: `pip3 install --upgrade code-review-graph`
